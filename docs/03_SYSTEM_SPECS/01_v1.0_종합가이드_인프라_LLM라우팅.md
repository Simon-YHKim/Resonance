# LLM 기반 GPS 텍스트 MMORPG — 비개발자(Vibe 코딩) 1인 제작자를 위한 종합 실행 가이드

> 본 문서는 2026년 4월 기준, 모바일 앱 + PWA 동시 지원, GPS 탐험형, 채팅 기반 LLM 전투/대화 진행 텍스트 MMORPG를 **자체 서버 없이 무료 티어로 시작**하고, **단계적으로 수익화**하기 위한 실행 가이드입니다. 비개발자(제조 엔지니어, Vibe 코딩 가능 수준)도 따라할 수 있게 구성했습니다.

## 0. 한 줄 결론(Executive Summary)

- **추천 1순위 스택:** **Capacitor(웹 코드 → Android APK + PWA) + Cloudflare Workers + Durable Objects(SQLite-backed) + D1 + KV + Cloudflare Turnstile + AI Gateway** 위에서 LLM은 **Gemini 2.5 Flash-Lite를 기본 라우팅**, 깊이 있는 대화/캐릭터 생성에는 **Claude Haiku 4.5**, 안전망으로 **GPT-5 mini**를 혼합 사용.
- **무료 운영 한계:** 캐릭터당 평균 5턴/일 LLM 호출 기준, **약 1,000~3,000 DAU**까지는 광고/구독 없이도 무료 티어 + LLM 호출비(월 $20~$50) 정도로 버틸 수 있음.
- **수익화 트리거:** DAU 500 또는 월 LLM 비용 $20 초과 시 **광고 시청 → 추가 행동권** 모델을 먼저 도입, DAU 2,000 또는 ARPDAU $0.05 가시화 시 **월 구독(예: 월 4,900원, 모델 업그레이드/컨텍스트 증가)** 추가.
- **가장 큰 리스크 3가지:** (1) LLM 호출 폭주(어뷰저 1명이 월 $1,000+ 발생 가능), (2) PWA의 백그라운드 GPS 불가 — 네이티브 앱 필수, (3) 닉네임 기반 캐릭터 생성의 욕설/저작권/PII 오용. 본 가이드 §6에서 모두 다룹니다.

- 위 라우팅 표 적용 시: ~**$0.7/일 = $20/월** (97% 절감)

### 1-3. 프롬프트 캐싱 — 반드시 켜야 하는 기능

LLM이 매 턴 받는 텍스트의 70% 이상은 "월드 룰북", "캐릭터 시트", "최근 대화 5턴" 같은 **반복 컨텍스트**입니다.

- **Anthropic(Claude):** `cache_control: {type: "ephemeral"}` 명시 필요. 5분 TTL은 쓰기 1.25× / 읽기 0.10× (1회 캐시 히트만 나도 본전), 1시간 TTL은 쓰기 2× / 읽기 0.10×. **최소 캐시 블록: Haiku 4.5는 4,096토큰**, Sonnet 4.6는 2,048토큰.
- **OpenAI(GPT-5 mini):** **자동** 캐싱(설정 불필요). 캐시 읽기 = 입력가의 10~25% (모델별).
- **Google(Gemini 2.5):** 2.5 Pro/Flash는 **암묵적 자동 캐싱** 지원. 별도 코드 거의 불필요.

**적용 효과 예시:** 시스템 프롬프트 8K 토큰 + 캐릭터 시트 2K 토큰 = 10K 토큰을 1,000번 호출 시
- 캐싱 X: 10K × 1,000 × $1/M = **$10/일** (Haiku 기준)
- 캐싱 O (Haiku 4.5): 첫 쓰기 $0.0125 + 999회 읽기 $1.0 = **약 $1.0/일 (90% 절감)**

### 1-4. 비용 절감을 위한 실전 패턴 7가지

1. **모델 라우팅(위 표):** 가장 강력. 95% 절감 가능.
2. **프롬프트 캐싱:** 시스템 프롬프트·월드 룰을 캐시.
3. **출력 토큰 상한(`max_tokens`)을 강제:** 전투 1턴 응답을 250토큰 이하로 제한. 출력 단가가 입력의 5배라 가장 큰 절약 포인트.
4. **사전 생성 콘텐츠 혼합:** 자주 쓰이는 몹 묘사 50종을 미리 LLM으로 1회 생성 → KV에 저장 → 일치 시 LLM 호출 없이 바로 사용.
5. **요약 압축(rolling summary):** 대화가 10턴 넘으면 직전 요약본 + 최근 3턴만 전송.
6. **구조화 출력(JSON 스키마):** 자유 산문 대신 `{"hp": -12, "loot": ["철검"], "narration": "..."}` 형식으로 출력 → 토큰 30% 감소 + 파싱 안정성.
7. **Cloudflare AI Gateway 경유:** 동일 입력에 대해 응답 캐시 자동 처리, 분석/속도 제한/예산 캡 무료 제공.

## 2. 서버리스 백엔드 — Cloudflare 스택 1순위 추천

### 2-1. 핵심: Cloudflare Workers Free + Durable Objects(SQLite) GA = MMORPG에 최적

2026년 1월부로 **SQLite-backed Durable Objects가 Workers Free 플랜에서 GA(정식)**가 되었습니다. 이건 MMORPG에 결정적인 변화입니다.
- **Durable Object 1개 = 1개의 "방" 또는 "월드 셀".** 같은 GPS 격자에 있는 플레이어들을 자동으로 같은 DO로 라우팅하면 **WebSocket 실시간 채팅·전투 진행이 무료**로 됩니다.
- **WebSocket Hibernation API**: 유저가 입력 안 할 땐 메모리에서 잠자기 → 듀레이션 과금 없음. 100명이 같이 채팅해도 메시지 없을 때 비용 0.

### 2-2. Cloudflare 무료 티어 한도 (2026년 4월 기준 정확 수치)

| 자원 | 무료 한도 (일·월) | 초과 시 |
|---|---|---|
| **Workers 요청** | **100,000 req/일** (UTC 자정 리셋) | 1027 에러로 fail (요금 X) |
| **Workers CPU 시간** | 무료 플랜 1요청당 10ms (유료 30s) | 에러 |
| **D1 (SQL) 행 읽기** | 5M 행/일 | 에러 |
| **D1 행 쓰기** | 100K 행/일 | 에러 |
| **D1 스토리지** | 5GB / 계정 | 쓰기 차단 |
| **Durable Objects(SQLite)** | 5GB 총 스토리지 / 계정, DO당 1GB | 쓰기 차단 |
| **DO 요청** | Workers와 통합 (100K/일 공유) | 에러 |
| **KV 읽기/쓰기** | 100K read / 1K write per day | 에러 |
| **R2(객체 스토리지)** | 10GB 저장 + 1M Class A op + 10M Class B op /월 | 에러 |
| **Cloudflare Turnstile** (캡차 대체) | **완전 무료, 무제한** | — |
| **AI Gateway 로그** | 100K logs/월 | 로그 누락(서비스는 동작) |
| **Cloudflare Pages** | 무제한 요청, 빌드 500/월 | — |

> 100K req/일 = **유저 1명당 100회 호출 시 1,000 DAU**까지 무료. WebSocket 메시지는 **20:1 비율로 청구** → 100만 메시지 = 5만 요청으로만 카운트되므로 채팅 게임에 매우 유리.

### 2-3. Workers 유료 전환 ($5/월부터)

- Workers Paid: $5/월 베이스에 10M 요청·30M CPU-ms 포함. 그 이상은 100만 요청당 $0.30, 100만 CPU-ms당 $0.02.
- DO 듀레이션: 활성 시간만 과금($12.50/M GB-sec ≈ 작은 채팅 게임은 월 1~3달러).
- **데이터 송신(egress) 무료** — Cloudflare의 핵심 경쟁력. AWS 대비 큰 절감.

### 2-4. 대안 플랫폼 비교 (Cloudflare가 안 맞을 때만)

| 플랫폼 | 무료 티어 | 장점 | 단점 (게임 관점) |
|---|---|---|---|
| **Cloudflare** (1순위) | 100K req/일, DO 5GB | 실시간 DO, 송신 무료, AI Gateway, Turnstile | KV 쓰기 1K/일 빡빡 |
| **Supabase** | DB 500MB, 50K MAU, 1주 비활성 시 자동 일시정지 ❗ | Postgres+Auth+Realtime 일체형, SQL 익숙 | 무료 프로젝트 자동 정지 — **출시용 부적합**, $25/월 Pro부터 안정 |
| **Firebase** | Spark 무료, Firestore 50K read/일 | 모바일 SDK 성숙 | 채팅처럼 fan-out 많은 워크로드에서 read 비용 폭주 |
| **Convex** | 함수 호출 1M/월 무료 | 실시간 reactivity, TS 친화 | 한국 리전 없음, 락인 |
| **Vercel** | Hobby 무료 (개인) | Next.js 통합 | 상업 사용 제한, WebSocket 없음 |

### 2-5. 권장 데이터 모델

| 데이터 | 저장소 | 사유 |
|---|---|---|
| 캐릭터 시트(닉네임으로 생성된 결과) | **D1**, JSON 컬럼 | 영구·관계형 쿼리 |
| 인벤토리·경험치·스킬 | **D1** | 트랜잭션 보장 |
| 활성 전투/대화 상태(임시) | **Durable Object SQLite** | 동시성·실시간 |
| 같은 GPS 셀의 플레이어 목록 | **Durable Object** (cell ID = DO 이름) | 자연스러운 샤딩 |
| 닉네임 → 캐릭터 결과 캐시 | **KV** (1년 TTL) | 동일 닉네임 재생성 방지(LLM 비용 절감) |
| 자주 쓰이는 몹 묘사·이벤트 텍스트 | **KV** | 조회 빈번, 쓰기 적음 |
| 위치 기반 쿼리(반경 내 스폰) | **D1** + R-tree 가상 테이블 또는 H3 헥스 인덱스 | SQLite 표준 기법 |
| 이미지·오디오(미래) | **R2** | 송신 무료 |

### 2-6. 인증

- **Clerk Free Tier**: 10,000 MAU 무료, 소셜 로그인 즉시. 1순위.
- **Supabase Auth**: Cloudflare 백엔드여도 Auth만 빌려와 사용 가능. 50K MAU 무료.
- **Cloudflare Access** + Email OTP: 가장 가볍지만 게임 UX엔 부족.

## 3. 모바일 앱 + 웹 동시 지원 — Vibe 코딩 친화적 접근

### 3-1. 결론: **Capacitor(또는 Expo) + 단일 React/Vue PWA 코드베이스**

비개발자가 ChatGPT/Claude/Cursor에 코드를 시키는 "Vibe 코딩" 환경에서는 **에이전트가 가장 잘 다루는 스택 = 웹 표준 스택**입니다. Flutter(Dart)보다 React/TS 코드 생성 품질이 훨씬 안정적.

| 옵션 | 학습 난이도 | 단일 코드 → 모바일+웹 | 백그라운드 GPS | LLM 코드 생성 품질 | 추천도 |
|---|---|---|---|---|---|
| **Capacitor + React + Vite** | 낮음 | ✅ Android APK + iOS + PWA | ✅ (커뮤니티 플러그인) | ★★★★★ | **1순위** |
| Expo (React Native) | 중간 | ✅ + Expo Go로 실시간 테스트 | ✅ (expo-location background) | ★★★★ | 2순위 |
| 순수 PWA만 | 낮음 | ✅ 웹만 | ❌ **백그라운드 위치 추적 불가** | ★★★★★ | MVP 검증용만 |
| Flutter | 높음 | ✅ + 웹 약함 | ✅ | ★★★ (Dart는 LLM이 React보단 약함) | 비추 |
| Tauri Mobile | 높음 (Rust) | ✅ | 미성숙 | ★★ | 비추 |

### 3-2. PWA의 결정적 약점: 백그라운드 GPS

- **PWA(브라우저)는 백그라운드(앱 닫힘/잠금)에서 위치 추적이 불가능**합니다. iOS Safari는 특히 엄격.
- 포켓몬 GO 스타일 "걸으면 알이 부화" 같은 UX는 **반드시 네이티브 빌드(APK/IPA) 필요**.
- 따라서: **PWA는 "GPS가 없거나 권한 거부 시 캐릭터 직접 컨트롤 모드" 전용**, 본격 GPS 탐험은 Android APK로 제공.

### 3-3. 권장 GPS 라이브러리

- **Capacitor 공식 `@capacitor/geolocation`**: 포그라운드 위치만. 웹에서도 동일 API.
- **`@capacitor-community/background-geolocation`** (무료) 또는 **`@transistorsoft/capacitor-background-geolocation`** (유료, 가장 견고): 백그라운드 추적·지오펜스·배터리 최적화.
- 권장: 출시 초기엔 무료 커뮤니티 버전 → DAU 늘면 Transistorsoft($조금) 도입.

### 3-4. 빌드·배포 비용

| 항목 | 비용 | 비고 |

| **Android Play Store 등록** | $25 일회성 | 필수 |

## 4. 수익화 모델 — GPS 게임 벤치마크 + LLM 비용 손익분기

### 4-1. AdMob/리워드 광고 eCPM 실측 (2024-2025)

| 광고 형식 | 글로벌 평균 eCPM | 미국 게임 eCPM | 완료율 |
|---|---|---|---|
| 배너 | $0.30~$0.70 | $0.45 (iOS) / $0.68 (Android) | — |
| 인터스티셜 | $5~$10 | $14.32 (iOS US) | 60~70% |
| **리워드 비디오** | **$8~$18** | **$15~$30** ($19.63 iOS / $16.49 Android Q4'24 US) | **>95%** |
| 오퍼월 | $200~$1,500 | (게임만 가능) | — |

**핵심 발견:** 게임 카테고리 리워드 비디오는 **다른 카테고리 대비 20~30% 높은 eCPM**. 사용자가 "보상을 위해 자발적으로" 시청하므로 UX 마찰도 최소.

### 4-2. LLM 비용 vs 광고 수익 손익분기 계산

**가정:**
- 평균 호출당 (라우팅 적용) **$0.001** (1K 입력 + 0.4K 출력 in Flash-Lite)
- 미국 유저 리워드 비디오 1회 = **$0.018** (eCPM $18 / 1,000)
- 한국 유저 리워드 비디오 1회 ≈ **$0.008** (eCPM ~$8)

→ **광고 1회 시청으로 한국 유저 약 8회 LLM 호출분 비용을 충당.** 미국 유저는 약 18회.

**모델 1: "행동 토큰" 시스템**
- 신규 유저에게 일일 무료 행동 5회 → 다 쓰면 30초 광고로 +5회 또는 광고 1회 = +1회 (보상 비례).
- 광고 시청률 50%, DAU 1,000 가정 시: 1,000 × 0.5 × 1 광고 × $0.008 = **$4/일 = $120/월** 광고 수익.
- 동 기간 LLM 비용: 1,000 × 10회 × $0.001 = **$10/일 = $300/월**
- → 광고만으론 부족, **구독 또는 IAP로 격차 보완 필요**.

## 5. LLM 비용 폭주 방지 (1인 개발자 생존 가이드)

### 5-1. 다층 방어선 7단계

1. **Cloudflare Turnstile (무료, 무제한)** — 회원가입 시 봇 차단. invisible 모드로 UX 영향 0.
2. **사용자별 rate limiting** — Workers의 `cf.duration` + KV 또는 DO 카운터로:
   - 무료: 시간당 10회, 일 30회 (광고로 +α)
   - 모험가: 시간당 60회, 일 무제한(단, 분당 5회 burst 제한)
3. **글로벌 예산 캡** — Cloudflare AI Gateway에 일일/월 USD 한도 설정. 도달 시 자동 fallback to "사전 생성된 응답" 또는 정중한 점검 메시지.
4. **`max_tokens` 강제** — 모든 호출에 `max_tokens: 300` 미만 설정 (보스전만 800).
5. **컨텍스트 윈도우 제한** — 캐릭터 시트 1.5K + 최근 5턴 요약 1K + 현재 입력 0.5K = **3K 입력 상한**.
6. **콘텐츠 사전 생성 ("Procedural + LLM 하이브리드")** — 몹 50종 × 첫 조우 멘트 5개 = 250개를 출시 전에 LLM으로 1회 생성하여 KV에 저장. 첫 조우 시 LLM 호출 0회.
7. **모니터링/알림** — Cloudflare Analytics + AI Gateway Cost Dashboard. 일일 비용 $5 초과 시 본인 휴대폰에 SMS(Twilio 무료 평가판이나 Resend 이메일 무료).

### 5-2. 닉네임 → 캐릭터 생성의 특수 보호

이 기능은 어뷰저 1순위 표적입니다 (욕설/저작권 캐릭터/개인정보 입력).
- **사전 필터:** GPT-5 Nano로 "이 닉네임이 욕설·실명·정치인·저작권 침해 캐릭터 이름인가?" 분류 ($0.00004/회).
- **결과 캐싱:** 같은 닉네임 → KV에서 재생성 차단 (LLM 비용 0). 단, 사용자당 1회만 무료 생성, 이후 ₩2,500 재생성권.
- **정책 룰북을 시스템 프롬프트에 명시:** "어떤 입력이든 PG-13 이내, 실존 인물 거부".

### 5-3. 부정 사용 패턴별 대응

| 패턴 | 탐지 | 대응 |
|---|---|---|
| 1초 간격 매크로 호출 | DO에서 사용자별 마지막 호출 timestamp | 429 응답 + 30초 쿨다운 |
| 같은 IP에서 100계정 생성 | Turnstile + Workers IP 카운터 | 가입 거부 |
| 매번 GPS 좌표 점프 (워딩) | 직전 좌표와 시속 계산 (>200km/h이면 차단) | 경고 → 정지 |
| 의도적으로 긴 입력 | Workers 단계에서 사용자 입력 토큰 카운트(`tiktoken`) | 1.5K 초과 시 거부 |

## 9. 주의사항 및 미확정 영역

- **Apple App Store는 Phase 3 이후 검토:** iOS는 "AI 가챠"·"확률형 콘텐츠"에 까다롭고, $99/년 비용 + 30% 수수료 + 심사 리스크가 큽니다. Android 우선 출시 권장.
- **개인정보·아동 보호:** 한국 게임법상 만 14세 미만 회원가입은 법정대리인 동의 절차 필요. Clerk나 자체 약관에서 14+로 일단 제한 추천.
- **위치정보 처리방침:** 위치정보의 보호 및 이용 등에 관한 법률(LBS법) 신고 의무가 있을 수 있음. 게임형은 약식이지만 출시 전 확인 권장.
- **Cloudflare Durable Objects SQLite 스토리지 과금:** **2026년 1월 7일 이후 사용량부터 과금 시작** — 현재 무료 5GB(Free)/무제한(Paid) 한도 내라면 무료지만, 정책이 변경될 수 있으니 분기마다 확인.
- **Anthropic Opus 4.7 토크나이저 변경(2026.04.16 출시):** Opus 4.7은 같은 텍스트가 5~35% 더 많은 토큰으로 변환되는 새 토크나이저 사용. **본 게임은 Opus 사용 권장하지 않으므로 영향 없음**, 단, 기록용으로 명시.
- **GPT-5 mini의 reasoning 모드:** "high reasoning"은 TTFT 100초+가 보고됨. 게임에선 반드시 `reasoning_effort: "minimal"`로 호출.
- **광고 eCPM 수치는 미국·영국 기준이며 한국은 통상 50~70%** 수준. 본 가이드 손익분기 계산은 보수적으로 한국 기준값($8 eCPM)을 적용했습니다.
- **모든 가격은 2026년 4월 기준 공식 발표가**이며, LLM 가격은 분기마다 인하되는 추세이므로 매월 1회 재확인 권장(가장 빠른 변동: Google Gemini, DeepSeek).
