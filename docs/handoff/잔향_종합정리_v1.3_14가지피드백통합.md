# 제1부. 결정적 변화 5개 — 시스템 재구성

## 1.1 변화 1 — 모남 조각 = 카테고리별 스킬 트리 (11트리) + WoW 매크로

### 1.1.1 핵심 결정

```
Before (v1.2):
  모남 조각 = 마이페이지 추억 기념품 (정적 컬렉션)

After (v1.3):
  모남 조각 = 전투 스킬 (동적 자산)
           + 추억 기념품 (정적 의미)
           + 사용자 매크로 (커스텀 빌드)
```

### 1.1.2 11카테고리 스킬 트리 구조

각 카테고리가 *하나의 스킬 트리*. 모남 조각 (소)을 모으면 트리가 깊어짐.

| 트리 | 스킬 결 (예시) |
|---|---|
| 1. 점심·결정장애 | 우유부단의 무기화 — 적이 헤매게 함 |
| 2. 출퇴근·이동 | 흐름의 무기화 — 자기 위치 흐릿하게 |
| 3. 카공·집중 | 깊은 몰입 — 단일 적 강타 |
| 4. 회식·관계 | 적당한 거리감 — 적 그룹 분리 |
| 5. 야근·시간 | 잘린 저녁 — 광역 슬로우 |
| 6. 운동·자기관리 | 자기 돌봄 — 회복 + 이동속도 |
| 7. 만남·외로움 | 빈자리의 호출 — 적 끌어당김 |
| 8. 학습·시험 | 답을 모르는 무기화 — 적 혼란 |
| 9. 쇼핑·소비 | 만족 못 함 — 적 디버프 누적 |
| 10. 가족·집 | 멀리서 부르는 한 마디 — 광역 위로 |
| 11. 늦은 밤·불면 | 천장 보는 시간 — 무적 짧게 |

### 1.1.3 트리 깊이 시스템

```
[스킬 트리 깊이 — 카테고리별]

Tier 1 (조각 1~3개): 기본 스킬 1개
Tier 2 (조각 4~7개): 변주 스킬 2개 중 선택
Tier 3 (조각 8~15개): 강화 스킬 3개 중 선택
Tier 4 (조각 16개+): 궁극기 스킬 1개

총 11트리 × 4단계 = 약 44개 스킬 슬롯
사용자가 11트리 동시 진행 가능 (특화 또는 균형 빌드)
```

### 1.1.4 WoW 매크로 시스템

```
[매크로 등록 형식]

/매크로명 = "[모남: 사용 카테고리] [상황: 컨텍스트] [행동: 액션]"

[예시]
/황당발언 = "[모남: 오늘 못한 한 마디] 
              [상황: 적의 발언 직후] 
              [행동: 무관한 질문으로 화제 전환]"

/조용히사라짐 = "[모남: 적당한 거리감 + 잘린 저녁] 
                  [상황: 적이 가까이] 
                  [행동: 시야에서 천천히 빠져나옴]"

[전투 중 사용]
the Named가 입력창에 "/황당발언" 한 단어 입력
→ LLM이 등록된 프롬프트 + 현재 컨텍스트 + 모남 트리 효과 적용
→ 액션 생성 + 효과 발동
```

**매크로 시스템의 의미**: the Named가 *자기만의 전투 페르소나*를 빌드. 1만 사용자 = 1만 매크로 라이브러리. 진짜 1인 1게임.

### 1.1.5 사용 방식 — 텍스트 + 매크로 통합

```
[일반 텍스트 입력 (자유)]
the Named: "오늘 못한 한 마디 사용 — 갑자기 — 황당한 질문"
→ LLM이 컨텍스트 기반 결과 생성. 실시간 변주.

[매크로 사용 (등록된 프롬프트)]
the Named: "/황당발언"
→ 등록된 프롬프트 호출. 빠른 사용.

[하이브리드]
the Named: "/황당발언 + 메뉴 고민 추가"
→ 매크로 + 즉석 추가. 베테랑 사용.
```

### 1.1.6 다양성 반영 — Q2-2 결정 적용

Simon이 *카테고리별 트리(a) + 다양성 반영*을 선택. 다양성을 박는 방법:

```
[다양성 1: 닉네임 컨텍스트 영향]
- 직장인 the Named의 *오늘 못한 한 마디* = 회사 회의 톤
- 대학생 the Named의 *오늘 못한 한 마디* = 동아리방 톤
- 같은 카테고리, 다른 컨텍스트, 다른 LLM 응답

[다양성 2: 모남 조각 자체의 변주]
- "오늘 못한 한 마디" 1개가 아님
- "오늘 회의에서 못 한 한 마디" / "오늘 부모님께 못 한 한 마디" / 
  "오늘 배우자에게 못 한 한 마디" — 같은 카테고리 안에 변주
- LLM이 컨텍스트에서 추출

[다양성 3: 시간대·헥스 영향]
- 새벽 사용 / 점심 사용 / 야근 사용
- 모남 조각 사용 결과가 시간대마다 미세하게 다름
```

이 셋이 박히면 — 11카테고리지만 실제 사용 시 *수천 개 변주*. Simon의 *바리에이션 반영* 결정 충족.

### 1.4.3 스토리 필수 카테고리 자연 노출 구조

스토리 진행에 *반드시* 박혀야 할 카테고리:

| 챕터 | 필수 카테고리 | 노출 시점 |
|---|---|---|
| 1장 | 늦은밤·불면 (자기정당화 모놀로그 호응) | 1장 1-2 첫 길거리 몹 |
| 1장 | 출퇴근 또는 회식 (보스 1 빌드업) | 1장 1-3 ~ 1-4 |
| 2장 | 운동·만남 (보스 2 한강 빌드업) | 2장 2-3 |
| 2장 | 학습·시험 (보스 3 학원가 빌드업) | 2장 2-6 |
| 3장 | 가족·집 (보스 4 어린시절 빌드업) | 3장 3-2 |
| 3장 | 결혼·운명 (성인의 책임 환기) | 3장 3-10 |

이 카테고리들은 — *LLM이 자유 변주*해도 *반드시 노출*되도록 강제. 그 외 카테고리는 LLM 자유.

```

  - 사용자 JSON (직업·환경·키워드)
  - 진행 챕터 + 필수 카테고리 (해당 시)
  - 최근 5분 트렌드 키워드
  - 헥스별 고유 해시태그
  
다음 11가이드라인 중 하나에 부합하거나, 새 카테고리 자유 생성:
[11카테고리 리스트]

스토리 필수 카테고리가 *오늘 미노출*이면 우선 생성."
```
