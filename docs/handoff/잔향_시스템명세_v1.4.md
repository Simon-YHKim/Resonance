## 1.1 데이터 구조 — D1 테이블

### 1.1.1 user_modaem_collection (모남 조각 영구 컬렉션)

```sql
CREATE TABLE user_modaem_collection (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  modaem_text TEXT NOT NULL,           -- "오늘 못한 한 마디의 한 조각"
  category TEXT NOT NULL,              -- "11카테고리 중 하나" or "추가카테고리"
  size TEXT NOT NULL,                  -- "small" or "large"
  source_type TEXT NOT NULL,           -- "street_mob", "boss", "mini_event"
  source_id TEXT,                      -- 처치한 몹·보스 ID
  acquired_at INTEGER NOT NULL,        -- timestamp
  childhood_memory TEXT,               -- v1.3 14번 — 어린 추억 LLM 생성
  hex_id TEXT,                         -- 어디서 획득
  context_snapshot_json TEXT,          -- 획득 당시 컨텍스트 (시간·날씨·트렌드)
  
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX idx_modaem_user_category ON user_modaem_collection(user_id, category);
CREATE INDEX idx_modaem_user_acquired ON user_modaem_collection(user_id, acquired_at DESC);
```

### 1.1.2 user_skill_tree (스킬 트리 진행도)

```sql
CREATE TABLE user_skill_tree (
  user_id TEXT NOT NULL,
  category TEXT NOT NULL,              -- 11카테고리 중 하나
  total_shards INTEGER NOT NULL DEFAULT 0,  -- 누적 조각 수
  current_tier INTEGER NOT NULL DEFAULT 0,  -- 0=비활성, 1~4
  selected_skills_json TEXT,           -- {"tier1": "skill_id", "tier2": "skill_id", ...}
  total_uses INTEGER NOT NULL DEFAULT 0,
  last_use_at INTEGER,
  
  PRIMARY KEY (user_id, category),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 1.1.3 user_macro (매크로 등록)

```sql
CREATE TABLE user_macro (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  macro_name TEXT NOT NULL,            -- "/황당발언" 형식, 슬래시 포함
  prompt_template TEXT NOT NULL,       -- 등록된 프롬프트
  used_categories_json TEXT,           -- 사용하는 모남 카테고리 리스트
  use_count INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  last_used_at INTEGER,
  
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE (user_id, macro_name)
);

CREATE INDEX idx_macro_user_use ON user_macro(user_id, use_count DESC);
```

### 1.3.2 매크로 사용 — 전투 중

```
[전투 화면 입력창]

플레이어 입력:
"/황당발언"

→ 시스템:
1. 등록된 매크로 검색 (user_macro 테이블)
2. 매크로 프롬프트 + 현재 컨텍스트로 LLM 호출
3. 사용된 모남 카테고리의 스킬 효과 적용
4. LLM 응답 + 효과 동시 표시
5. use_count 증가
```

### 1.3.3 자유 텍스트 vs 매크로 — 둘 다 가능

```
[자유 텍스트]
"오늘 못한 한 마디 사용 — 회의실에서 갑자기 이상한 질문"
→ 매번 LLM이 처리. 신선함. 토큰 소모 ↑

[매크로]
"/황당발언"
→ 등록된 프롬프트 활용. 빠름. 토큰 소모 ↓ (캐싱 가능)

[하이브리드]
"/황당발언 + 메뉴 추천도"
→ 매크로 + 즉석 추가. 베테랑 사용.
```

## 1.4 다양성 반영 — Q2-2 결정 적용

### 1.4.1 닉네임 컨텍스트 영향

```
[직장인 the Named의 "오늘 못한 한 마디" 사용]
LLM 응답 결: 회의실 톤
"갑자기 — 회의 중 무관한 농담을 던집니다.
 *과장님, 점심 뭐 드시나요?* 
 회의실에 어색한 침묵이 흐릅니다. 적이 당황합니다."

[대학생 the Named의 같은 스킬 사용]
LLM 응답 결: 동아리방 톤
"갑자기 — 회의 중 다른 화제 던집니다.
 *어, 그거 어제 본 영화 같지 않아요?* 
 동기들이 *얘 또 시작이네* 표정. 적이 헷갈립니다."

[주부 the Named의 같은 스킬 사용]
LLM 응답 결: 가족 식탁 톤
"갑자기 — 무관한 안부를 묻습니다.
 *어머님, 점심은 드셨어요?* 
 시어머니가 *그게 지금 중요하니* 표정. 적이 당황합니다."
```

### 1.4.2 모남 조각 자체의 변주

```
"오늘 못한 한 마디" 카테고리 안에 변주:
- "오늘 회의에서 못한 한 마디"
- "오늘 부모님께 못한 한 마디"
- "오늘 배우자에게 못한 한 마디"
- "오늘 친구에게 못한 한 마디"
- "오늘 자기 자신에게 못한 한 마디"

LLM이 컨텍스트에서 추출. 각 변주가 다른 LLM 응답.
```

### 1.4.3 시간대·헥스 영향

```
같은 "오늘 못한 한 마디" 사용:
- 점심 사용: 회의실·식당 톤
- 야근 사용: 빈 사무실·업무 톤
- 새벽 사용: 침대·내면 톤

LLM 응답이 시간대에 맞춰 미세 조정.
```

이 셋이 박히면 — 11카테고리지만 *수천 가지 변주*. 사용자마다·시간마다·헥스마다 다른 경험.

## 2.1 닉네임 분석 LLM 프롬프트 v1.4

### 2.1.1 시스템 프롬프트

```
당신은 게임 *잔향(Resonance)*의 닉네임 분석가입니다.
사용자가 입력한 닉네임에서 다음을 추론해주세요:

1. 직업·신분 (직장인/대학생/대학원생/주부/은퇴자/프리랜서/학생/기타)
2. 추정 연령대 (10대/20대/30대/40대/50대+)
3. 추정 환경 (사무실/캠퍼스/연구실/집·동네/공방/기타)
```

### 4.2.2 집계 뷰

```sql
-- 사용자별 일일 토큰 소비
CREATE VIEW user_daily_tokens AS
SELECT 
  user_id,
  DATE(timestamp/1000, 'unixepoch') as date,
  SUM(input_tokens + output_tokens) as total_tokens,
  SUM(cost_usd) as total_cost,
  COUNT(*) as call_count
FROM llm_usage_log
GROUP BY user_id, DATE(timestamp/1000, 'unixepoch');

-- 무료 vs 프리미엄 비교
CREATE VIEW free_vs_premium_avg AS
SELECT 
  is_premium,
  AVG(daily_tokens) as avg_daily_tokens,
  AVG(daily_cost) as avg_daily_cost
FROM (
  SELECT 
    user_id,
    is_premium,
    SUM(input_tokens + output_tokens) as daily_tokens,
    SUM(cost_usd) as daily_cost
  FROM llm_usage_log
  WHERE timestamp > strftime('%s', 'now', '-7 days') * 1000
  GROUP BY user_id, DATE(timestamp/1000, 'unixepoch')
);
```

## 4.3 스테미나 포션 가격 알고리즘

```
[원가 계산]
1포션 = 30분 추가 플레이
30분 평균 LLM 호출 = 6회 (5분에 1회)
1호출 평균 토큰 = 1,800 입력 + 200 출력
1호출 평균 비용 (Flash-Lite 기준) = $0.00026

1포션 원가 = 6 × $0.00026 = $0.00156 = ₩2

[마진 가이드]
무료 사용자 평균 1주일 1포션 구매 가정
→ 월 4포션 × 가격 = 사용자당 매출

[제안 가격]
1포션 = ₩400 (마진 200배)
3포션 패키지 = ₩1,000 (할인)
10포션 패키지 = ₩3,000 (대량)
무제한 1일 = ₩2,500
무제한 1주 = ₩9,800
무제한 1개월 = ₩29,000

[조정 트리거]
출시 후 1개월 데이터 수집:
- 평균 토큰이 1,800보다 ↑ → 가격 ↑
- 무료 사용자 이탈률이 ↑ → 무료 한도 ↑
- 프리미엄 전환율이 < 5% → 가격 ↓ 또는 가치 ↑
```

```sql
CREATE TABLE violation_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  violation_type TEXT NOT NULL,
  severity TEXT NOT NULL,
  input_text TEXT,  -- 차단된 입력 (개인정보 검토용)
  timestamp INTEGER NOT NULL,
  action_taken TEXT NOT NULL,  -- "warning" / "stamina_loss" / "input_block" / "session_pause" / "account_review"
  
  FOREIGN KEY (user_id) REFERENCES users(id)
);

[누적 룰]
```
