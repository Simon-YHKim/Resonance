## ⚠️ 주의사항

### 1. v2.x 기획서들 안의 *결정 일부*는 v1.2에서 진화했어요

특히 다음 항목들은 v2.x 원본 기획서를 *그대로 따르면 안 되고*, v1.2의 재해석을 따라야 합니다:

| 항목 | v2.x 원본 | v1.2 진화 |
|---|---|---|
| 보스 = 적 | v2.3에 그렇게 표현 | **보스 = 모남/꿈자국 조각** (적 X) |
| 분기 엔딩 = 동행/작별만 | v2.3에 그렇게 결정 | **6엔딩 = 동행/작별 × 개화/여전/항로** |
| 기억의 조각 | v2.2에서 명명 | **모남 조각 + 꿈자국 조각 두 결** |
| 종합 기여도 4축 | v2.3에서 결정 | **3축 (개화/여전/항로) 점수 메커니즘과 통합** |

### 2. 자체 세계관 명명 — v2.1 → v2.2에서 정정됨

v2.1에서는 *"잔향계 + 원향계"* 라고 명명했지만, **v2.2에서 *"잔향 + 원"* 으로 최종 확정**. v2.1 읽을 때 이 변경 유의.

### 3. v1.0 종합가이드 ≠ v1.0 종합정리

| 헷갈리지 마세요 |
|---|
| `03_SYSTEM_SPECS/01_v1.0_종합가이드` — *기획서 v1.0의 시스템 명세* |
| `05_ARCHIVE/05_종합정리_v1.0` — *변천사 추적 종합 정리 v1.0* |

이름이 비슷하지만 다른 문서. 시스템 명세를 보려면 03 폴더, 변천사를 보려면 05 폴더.

## 🌱 마무리 — Claude Code에게 한 마디

잔향은 — 코드도 중요하지만 **결과 톤이 더 중요한 게임**입니다. 시스템 명세를 정확히 구현하는 것만으로는 부족해요. 5대 영혼이 코드 안에 살아 있어야 합니다.

빌드하다가 *"이 코드가 위로하는 게임 톤에 맞나?"* 자문이 들면 — 멈추세요. 그 자문이 떠올랐다는 것 자체가 *틀렸다는 신호*일 가능성이 높습니다. 그럴 때 PROJECT_SUMMARY와 01_종합정리_v1.2를 다시 읽으세요.

잔향이 빌드되는 그날까지 — 5대 영혼을 잊지 마세요.
