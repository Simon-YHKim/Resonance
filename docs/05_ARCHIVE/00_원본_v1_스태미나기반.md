## 1. 게임 개요와 핵심 가치

### 1.1 한 줄 정의
**"네가 적은 닉네임이 너의 인생을 결정하고, 네가 사는 동네가 너의 던전이 되는 텍스트 MMORPG."**

### 1.2 3가지 게임 기둥(Pillars)

| 기둥 | 설명 | 차별화 포인트 |
|---|---|---|
| **운명(Fate)** | 닉네임을 적는 순간 LLM이 캐릭터의 성격·특기·약점·시작 스킬·말투를 생성한다. | 다른 RPG는 직업/스탯을 고르지만, 본 게임은 "내가 나도 모르게 적은 한 단어"가 운명이 됨 |
| **장소(Place)** | 내 집·내 회사·내 자주 가는 카페가 던전·NPC 거점·보스 출현지가 된다. | Pokémon GO는 위치를 단지 좌표로 쓰지만, 본 게임은 LLM이 그 장소의 분위기를 묘사 |
| **언어(Language)** | 모든 행동은 채팅으로 지시한다. "검을 뽑아 정중히 인사한다" 같은 자유 입력이 가능 | 텍스트 RPG는 많지만 GPS+MMO+LLM의 조합은 사실상 첫 시도 |

### 1.3 타깃 유저
- **Primary**: 25~40세 텍스트 RPG·머드 게임 향수 세대 + AI 챗봇(캐릭터봇) 매니아
- **Secondary**: 산책·러닝하면서 콘텐츠 즐기고 싶은 라이트 게이머
- **출시 지역**: 한국 1차 → 일본/대만 2차 (텍스트 콘텐츠 친화적 시장 우선)

### 1.4 차별화 / 안 하는 것
- ❌ 그래픽/3D/이미지 생성 (비용 폭주의 원인)
- ❌ PvP 실시간 전투 (서버 부담 + LLM 비동기 한계)
- ❌ 무제한 자유 채팅 NPC (스태미나로 게이트)
- ✅ 텍스트만으로 머릿속에 그려지는 묘사
- ✅ "내일도 와야 하는" 일일 루틴 게임 (스태미나 회복 모델)

## 2. 핵심 게임플레이 루프

### 2.1 매크로 루프 (일일)
```
[잠자기/일상] → [출근길·점심·퇴근길에 앱 열기] → [GPS로 주변 몹 확인] 
    → [스태미나 한도 내에서 1~3건 전투] → [전리품·경험치 획득] 
    → [길드 채팅 확인] → [앱 닫기] → [스태미나 회복 대기]
```
- 평균 세션: **3~5분 × 일 3~5회** (스태미나가 강제하는 분할 플레이)
- DAU > MAU의 비율이 자연스럽게 높아짐 (매일 스태미나 리셋)

### 2.2 마이크로 루프 (1전투, 약 90초)
```
[몹 발견 알림] → [몹 묘사 텍스트 (LLM/캐시)] → [행동 입력 채팅] 
    → [LLM이 결과 묘사 + 데미지/효과 계산] → [반복 또는 종료] 
    → [전리품·경험치] → [스태미나 차감 표시]
```

### 2.3 진행 동기 분석 (Bartle Type 대응)

| 유저 타입 | 본 게임의 동기 |
|---|---|
| **Achiever** | 레벨·도감·길드 랭킹·시즌 패스 |
| **Explorer** | 새 동네 가면 새 몹·새 NPC, 닉네임 별 스토리 분기 |
| **Socializer** | 길드 채팅, 공동 보스 레이드, 거래 |
| **Killer** | 길드전, 영토 점령(Phase 3+), 리더보드 |

## 3. 스태미나 시스템 — 게임의 척추

### 3.1 핵심 원칙: **1 스태미나 = LLM 비용 $0.0002 = 약 0.27원**

이 환율이 게임 전체 경제의 출발점입니다. 모든 행동 비용·구매 가격·광고 보상이 이 환율에서 도출됩니다.

**환율 도출 근거** (Gemini 2.5 Flash-Lite 기준):
- 평균 1턴 호출 = 1.5K 입력 + 0.4K 출력 = $0.000310
- 안전 마진 50% 추가 → **$0.0002 ~ $0.0005 / 1 스태미나**
- 보수적으로 **0.3원 / 스태미나** 로 가격 환산

### 3.2 스태미나 풀 (Stamina Pool)

| 등급 | 최대치 | 회복 속도 | 회복 완료 시간 |
|---|---|---|---|
| 무료 (Wanderer) | **100** | 1 / 6분 (10/시간) | 10시간 |
| 모험가 (Adventurer, ₩4,900/월) | 300 | 1 / 4분 (15/시간) | 20시간 |
| 전설 (Legend, ₩14,900/월) | 800 | 1 / 3분 (20/시간) | 40시간 |

> **설계 의도**: 무료 유저는 잠자고 일어나면 풀충, 모험가/전설은 더 오래 누적해서 "주말 한방 플레이" 가능. 회복 속도를 너무 빨리 잡으면 어뷰저가 24시간 매크로로 LLM 비용 폭주 가능.

### 3.3 행동별 스태미나 비용표

| 행동 | 스태미나 | LLM 호출 | 비고 |
|---|---|---|---|
| **이동 / GPS 이동** | 0 | 없음 | 스태미나 0 = 자유 탐험 |
| **몹 조우 묘사 (첫 텍스트)** | 1 | 캐시 우선 / 신규 몹만 LLM | KV 캐시 히트 시 LLM 0회 |
| **일반 공격 (검·주먹·마법)** | 3 | Flash-Lite 1회 | 짧은 결과 묘사 + 데미지 계산 |
| **스킬 사용** | 5 | Flash-Lite 1회 | 스킬 효과 묘사 |
| **자유 행동 입력** ("검에 키스하고 던진다") | 5 | Flash-Lite 1회 | 창의적 입력 LLM이 해석 |
| **NPC 일반 대화 (1턴)** | 3 | Flash-Lite 1회 | |
| **NPC 깊은 대화 / 퀘스트 협상** | 8 | Haiku 4.5 1회 | 캐릭터성 중요한 NPC |
| **보스전 1턴** | 10 | Sonnet 4.6 1회 | 큰 묘사 + 페이즈 처리 |
| **닉네임 → 캐릭터 생성** | 30 | Haiku 4.5 1회 | 또는 신규 회원 무료 1회 |
| **닉네임 재생성 (리롤)** | 50 또는 ₩2,500 IAP | Haiku 4.5 1회 | |
| **장비 감정** ("이 검은 어떤 검이지?") | 2 | Flash-Lite (캐시 가능) | |
| **로어/세계관 질문 (도서관 NPC)** | 4 | Flash-Lite + 캐시 | |
| **일일 길드 인사말 입력** | 0 | LLM 호출 없음 | 사회적 행동 무료화 |

### 3.4 일일 플레이 시뮬레이션

**무료 유저 평균 일일 사용:**
- 잡몹 전투 3회 × 평균 4턴 = 36 스태미나
- NPC 대화 1회 × 3턴 = 9 스태미나
- 자유 행동/실험 = 5 스태미나
- 합계: **약 50 스태미나** (남는 50은 다음날 누적 X, 풀 캡)
- 일일 LLM 비용: 약 **₩15 / 유저**

**모험가 구독자 평균 일일 사용:**
- 잡몹 8회 × 5턴 = 120 스태미나
- NPC 깊은 대화 2회 × 3턴 = 48 스태미나
- 보스전 1회 × 5턴 = 50 스태미나
- 합계: **약 218 스태미나**
- 일일 LLM 비용: 약 **₩75 / 유저** (구독료 ₩4,900/월 = 일 ₩163, **흑자 마진 ₩88/일**)

### 3.5 스태미나 충전 수단 (4가지)

| 수단 | 보상 | 일일 한도 | 비용 (유저 입장) | 비용 (개발자 입장) |
|---|---|---|---|---|
| **시간 회복** | 자동 | 무제한 (캡까지) | 무료 | 무료 |
| **광고 시청 (30초)** | +20 스태미나 | **5회/일 (총 +100)** | 무료 (시간) | 광고 수익 ₩40 vs LLM 비용 ₩30 = **₩10 흑자** |
| **소량 포션 IAP** | +50 스태미나 | 무제한 | ₩1,100 | LLM 원가 ₩15 → **마진 99%** |
| **중형 포션 IAP** | +200 스태미나 (보너스 +20) | 무제한 | ₩3,300 | 원가 ₩60 → 마진 98% |
| **대형 포션 IAP** | +500 스태미나 (보너스 +100) | 일 1회 | ₩6,600 | 원가 ₩150 → 마진 97% |
| **일일 출석 보상** | +30 스태미나 | 1회/일 | 무료 | 무료 (리텐션) |
| **7일 연속 출석** | +200 스태미나 | 1회/주 | 무료 | 무료 (강력한 리텐션 훅) |
| **첫 결제 보너스** | +1,000 스태미나 | 1회/계정 | ₩1,100~ 어떤 결제든 | 강력한 첫 결제 유도 |

### 3.6 스태미나 UI 표시 규칙
- 화면 상단 항상 표시: `⚡ 47 / 100  ⏱ 다음 회복까지 03:21`
- 스태미나 부족 시: 빨간색 + 진동 + 충전 옵션 모달
- 충전 모달은 광고 시청을 **첫 번째 옵션**으로 (유저 호의 + 수익 모두 ↑)

### 3.7 어뷰저 방지 추가 장치
- 한 행동에 스태미나 부족 시 **"미완료 상태 저장"** 후 다음 회복 때 이어서 (포기를 줄임)
- 동일 NPC 24시간 내 깊은 대화 횟수 = 5회 캡 (LLM 비용 폭주 차단)
- 자유 행동 입력 글자 수 = **최대 200자** (어뷰저가 5,000자 입력으로 토큰 폭주시키는 공격 방지)
- 일일 신규 캐릭터 생성 = IP당 3회

## 4. 닉네임 캐릭터 생성 시스템

### 4.1 핵심 메커니즘
유저가 닉네임을 입력하면 LLM(Claude Haiku 4.5)이 다음 7가지 항목을 생성합니다.

```yaml
입력: "잿빛고양이"
출력 (LLM이 30초 안에 생성):
  연상_키워드: ["회색", "은밀함", "도시 뒷골목", "고독", "민첩"]
  성격: "냉소적이지만 정 많은 외로운 도둑"
  말투: "...뭐, 어쩌라고. 그냥 따라와."
  시작_직업: "그림자 도적"
  시작_스탯:
    힘: 8
    민첩: 16
    지능: 12
    매력: 10
    운: 14  # 닉네임에 "고양이"라서 운이 높음
  시작_스킬: 
    - "은신": "그림자에 녹아 다음 한 턴 회피율 +50%"
    - "발톱 강타": "민첩 기반 일반 공격, 25% 추가 출혈"
  시작_장비:
    - "낡은 회색 단검"
    - "찢어진 후드"
  배경_스토리: "이름조차 잊은 도둑. 비 오는 골목에서 고양이 한 마리에게 빵을 나눠줬던 그날 이후, 그림자가 그를 따라다닌다."
  약점: "햇빛이 강한 정오에 모든 능력치 -10%"  # 흥미로운 핸디캡
  운명_훅(Fate Hook): "어딘가에 너와 같은 이름의 검은 고양이 보스가 있다."
```

### 4.2 LLM 프롬프트 설계 (시스템 프롬프트, 캐시됨)

```
당신은 "운명의 닉네임" 게임의 캐릭터 생성자입니다.

규칙:
1. 닉네임에서 연상되는 키워드를 5개 추출하세요.
2. 그 키워드 조합에서 가장 흥미로운 RPG 캐릭터를 만드세요.
3. 절대 평범한 "전사/마법사/도적" 직업이 아니라, "그림자 도적", "녹슨 검의 폭검사" 같은 구체적인 직업명을 만드세요.
4. 약점을 반드시 1개 부여하세요 (게임 다양성 확보).
5. "운명 훅" — 미래에 만날 보스나 NPC와의 연결고리를 1개 심으세요.
6. PG-13 이내. 욕설/실명/저작권 캐릭터 거부.
7. 한국어 닉네임 → 한국 판타지 톤, 영어 닉네임 → 서양 판타지 톤.

스탯 합계는 항상 60. 5개 스탯에 8~16 사이로 분배.

JSON 스키마로만 응답하세요:
{ "연상_키워드": [...], "성격": "...", "말투": "...", ... }
```

**캐싱 효과**: 위 시스템 프롬프트(약 1.5K 토큰)는 모든 캐릭터 생성에 동일 → Anthropic 명시적 캐싱으로 **첫 호출 외 90% 비용 절감**.

### 4.3 욕설/오용 방지 파이프라인

```
[유저 입력 닉네임] 
  → [GPT-5 Nano 필터: 욕설·실명·저작권 캐릭터·PII 분류] (₩0.05/회)
  → [통과 시] [KV 캐시 조회: 같은 닉네임 이미 있나?]
    → [있음] [기존 결과 재사용, LLM 호출 0]
    → [없음] [Haiku 4.5 호출] [결과 KV 저장 (TTL 1년)]
  → [실패 시] [정중한 거부 + 다른 닉네임 제안]
```

### 4.4 같은 닉네임 충돌 처리
- KV에 닉네임 → 캐릭터 결과 매핑 저장 (1년 TTL)
- 같은 닉네임 = 같은 운명 = 같은 캐릭터 (시드처럼 작동)
- 단, 게임 내 표시는 `잿빛고양이#0042` 같이 식별자 부여 (중복 허용)
- **재미 포인트**: "야, 너도 잿빛고양이야? 우리 같은 운명이네!" 같은 사회적 발견

### 4.5 리롤(Reroll) 정책
- 첫 생성: 신규 회원 1회 무료
- 2회 이후: **50 스태미나** 또는 **₩2,500 IAP**
- 리롤 시 LLM에 "이전 결과: ... 이번엔 다른 방향으로" 프롬프트 추가 → 다양성 확보

## 5. 전투 시스템

### 5.1 전투 흐름
```
1. 몹 조우 (GPS 또는 가상 이동) 
2. 몹 묘사 텍스트 (LLM 또는 캐시)
3. [전투 시작] 
   ├─ 유저 턴: 채팅 입력 ("검을 뽑아 베기")
   │   ├─ LLM이 행동 해석 → 데미지 계산 → 묘사 출력
   │   └─ 스태미나 차감
   ├─ 몹 턴: LLM이 몹 행동 결정 + 묘사
   ├─ HP 체크 → 종료 또는 다음 턴
4. 종료 시: 경험치, 골드, 전리품, 가능 시 스킬/장비 드롭
```

### 5.2 데미지 계산 (LLM과 결정론적 시스템의 하이브리드)

**핵심 결정**: LLM에게 데미지 숫자를 결정시키지 않습니다. LLM은 **묘사**, 시스템은 **숫자**.

```typescript
// LLM 출력 (JSON 강제)
{
  "narration": "잿빛고양이는 어둠 속에서 단검을 휘두른다. 차가운 칼끝이 고블린의 어깨를 스쳤다.",
  "action_type": "physical_attack",  // 시스템이 인식하는 분류
  "skill_used": null,
  "modifier": "critical_hit_attempt",  // 시스템이 해석할 수정자
  "narrative_flags": ["stealth_bonus"]  // 은신 상태 활용
}

// 시스템이 데미지 계산
damage = base_attack(STR, weapon) 
       × random(0.8, 1.2) 
       × (skill_used ? skill_multiplier : 1.0)
       × (critical_hit ? 2.0 : 1.0)
       × (stealth_bonus ? 1.5 : 1.0)
```

**장점:**
- LLM이 "갑자기 99999 데미지"를 출력해도 시스템이 무시 → 밸런스 보호
- LLM 출력 JSON이 망가져도 묘사만 빠지고 게임은 진행 가능 (안정성)

### 5.3 몹 등급별 LLM 모델 할당

| 몹 등급 | 사용 모델 | 1턴 비용 | 사용 빈도 |
|---|---|---|---|
| **잡몹 (1~3레벨 차이)** | Flash-Lite | ₩0.27 | 80% |
| **엘리트 (퀘스트 몹)** | Flash 또는 Haiku | ₩1 | 15% |
| **보스 (이름 있음)** | Haiku 4.5 | ₩3 | 4% |
| **레이드 보스 (길드)** | Sonnet 4.6 | ₩10 | 1% |

### 5.4 자유 행동 입력 처리

유저가 "검에 침을 뱉고 휘두른다" 같은 비표준 행동을 입력하면:

```
LLM 시스템 프롬프트 (요약):
"유저가 자유 행동을 입력했다. 다음 중 하나로 분류하라:
- 합리적 → 보너스/페널티 적용 묘사 + 일반 처리
- 창의적 → +20% 데미지 보너스 (재미 보상)
- 무의미/장난 → 묘사만 하고 데미지 없음, 몹의 비웃음
- 시도 불가 → 캐릭터가 시도하다 실패 묘사

JSON 출력하라."
```

**설계 의도**: 자유도를 보장하면서도 어뷰저("나는 신이다, 모든 적을 즉사시킨다")가 시스템을 깨지 못하게.

### 5.5 처치 시 보상

```
경험치 = 몹 레벨 × 10 × (처음_본_종이면 1.5)
골드 = 몹 레벨 × random(5, 15)
스태미나 환불 = 몹 레벨 ÷ 5 (큰 몹 잡으면 약간 회복) ← 흥미로운 보상 루프
전리품 = drop_table(몹_종, 럭_스탯)
스킬_드롭 = 매우 희박 (몹 종당 1~2개 고유 스킬)
```

**스태미나 환불**의 게임 디자인 효과: "어려운 적을 잡으면 더 놀 수 있다" → 도전 동기. 하지만 환불액 < 소비액으로 설정해서 무한 플레이 방지.

## 6. GPS 탐험 시스템

### 6.1 H3 헥스 그리드
- 세계를 H3 헥스(level 9, 약 174m 변 길이)로 분할
- 1 헥스 = 1 Durable Object (실시간 채팅·몹 풀 관리)
- 인접 헥스로 이동 시 새 DO에 연결
- 헥스 ID = `8a2a1072b59ffff` 같은 고유 ID로 글로벌 좌표 = 자연스러운 샤딩

### 6.2 몹 스폰 규칙

**확정적 스폰 (서버 부담 0):**
- 헥스 ID + 현재 시간(15분 단위 슬롯)으로 시드 → 의사난수로 몹 결정
- 같은 시간대 같은 장소 = 같은 유저들이 같은 몹 본다 → "공동 발견"
- LLM 호출 없이 정해진 몹 풀에서 추첨

**예시 알고리즘:**
```
seed = hash(hex_id + floor(timestamp / 900))  // 15분
mob_pool = get_mob_pool_by_zone(hex_id)  // 도시/주거/공원/강가/산
selected = mob_pool[seed % len(mob_pool)]
spawn_count = 1 + (seed % 3)  // 1~3마리
```

### 6.3 지역(Zone) 분류
헥스 ID를 OSM(OpenStreetMap, 무료) 데이터에 매핑하여 지역 타입 결정:

| Zone Type | 등장 몹 예시 | 특수 효과 |
|---|---|---|
| **도시 번화가** | 부랑자 모리스, 길거리 마술사, 술취한 기사 | NPC 출현률 ↑ |
| **주거지** | 옆집 고양이 정령, 길 잃은 강아지 정령 | 평화로운 이벤트 |
| **공원** | 나무 정령, 잠든 호랑이의 영혼 | 회복 NPC |
| **강가/해안** | 물의 어부, 익사한 영혼 | 낚시 미니게임 |
| **산/숲** | 늑대 무리, 산적 | 보스 출현 확률 ↑ |
| **상점가** | NPC 상인 (LLM 흥정 가능) | 쇼핑 가능 |
| **학교/도서관** | 도서 정령, 떠도는 박사 | 로어 NPC |
| **밤(22~05시)** | 모든 몹 등급 +1, 야행성 몹 추가 | 위험·보상 ↑ |

### 6.4 GPS 없는 모드 ("가상 이동")
PWA·실내·GPS 거부 시:
- 가상 캐릭터를 화면에서 방향키/탭으로 조작
- 현실 거리는 적용되지 않고, 가상 맵에서 헥스 이동
- **차이점**: 가상 모드는 몹 보상 80% (현실 GPS 보상이 우월하도록)
- 동일 가상 헥스에 다수 유저 동시 접속 가능 (DO가 정원 50명까지)

### 6.5 거리 측정 보안 (포켓몬GO식 GPS 스푸핑 차단)

```python
def validate_movement(user_id, new_lat, new_lng, timestamp):
    last = get_last_position(user_id)
    distance = haversine(last, new)  # km
    elapsed = (timestamp - last.timestamp).hours
    speed = distance / elapsed
    
    if speed > 200:  # km/h, 자동차도 무리
        flag_for_review(user_id)
        return False
    if speed > 60 and not is_whitelisted_route(user, last, new):  # 차량 주행
        require_captcha(user_id)  # Turnstile
    
    return True
```

### 6.6 특수 위치 — POI(Point of Interest)
- OSM 또는 자체 큐레이션으로 카페/공원/지하철역 등 등록
- POI = 던전 입구·NPC 거점·일일 보스 출현지
- 출시 후 **유저 제보**로 추가 가능 (커뮤니티 모더레이션)
- **수익화**: Phase 3 단계에서 동네 카페와 월 ₩50K 스폰서 계약 (해당 카페에 한정 보스/NPC 등장)

## 7. 성장 시스템

### 7.1 레벨과 경험치
- 레벨 1~50 (시즌 1)
- EXP 곡선: `next_level = level^2 × 100` (50레벨 도달까지 약 3개월 일평균 플레이)
- 레벨업 보상: 스탯 포인트 +3, 새 스킬 슬롯 1개

### 7.2 스킬 시스템

**스킬 획득 경로:**
1. **닉네임 시작 스킬**: 캐릭터 생성 시 1~2개 자동 부여
2. **레벨업 스킬 트리**: 5레벨마다 직업 트리에서 1개 선택 (LLM 미사용, 정해진 풀)
3. **드롭 스킬**: 몹 처치 시 매우 낮은 확률로 고유 스킬 책 드롭
4. **퀘스트 보상**: NPC 퀘스트 완료 시 (LLM이 캐릭터 행적 기반 맞춤 스킬 생성)

**스킬 데이터 구조:**
```yaml
skill_id: "shadow_step"
이름: "그림자 보법"
설명: "다음 한 턴 회피율 +50%, 자신의 다음 공격 +25% 데미지"
사용_스태미나: 5
쿨다운: 3턴
LLM_묘사_프롬프트: "사용자가 그림자 속에 녹아드는 모습을 1~2문장으로 묘사하라"
획득_조건: "잿빛 계열 닉네임 + 도적 직업 + 5레벨"
```

### 7.3 장비 시스템

**4슬롯 단순화** (텍스트 게임 UX 우선):
- 무기 / 방어구 / 장신구 / 부적

**장비 등급:**
- 일반(흰) / 고급(녹) / 희귀(파) / 영웅(보) / 전설(주황) / **운명(빨강, 닉네임 연관)**

**"운명" 등급의 특수 메커니즘:**
- 닉네임에서 연상되는 키워드와 매칭되는 장비 드롭 시 자동 "운명" 등급으로 승격
- 예: "잿빛고양이"가 "고양이 발톱"을 얻으면 운명급
- 사회적 발견의 즐거움 + 장비 도감 욕구 자극

### 7.4 평판/팩션 시스템 (Phase 2+)
- 5개 팩션: 왕국 / 도적 길드 / 학자회 / 자연 수호자 / 어둠의 결사
- 각 팩션 평판 -100 ~ +100
- NPC 행동·퀘스트·상점가 가격이 평판에 따라 변화
- LLM이 평판 정보를 시스템 프롬프트에 포함하여 NPC 반응 결정

## 9. 콘텐츠 시스템

| 몹 첫 조우 멘트 (50종 × 5변형) | 출시 전 일괄 생성 → KV 저장 | $0 (런타임) |

| NPC 정해진 대사 (퀘스트 진행) | 출시 전 작성 | $0 |
| NPC 자유 대화 | 실시간 LLM (Haiku) | ₩1/턴 |
| 캐릭터 생성 | 실시간 LLM (Haiku, 캐싱) | ₩12/회 |
| 보스 페이즈 묘사 | 실시간 LLM (Sonnet) | ₩3/턴 |
| 시즌 이벤트 스토리 | 출시 전 작성 + 분기점만 LLM | ₩50/유저 |

### 9.2 출시 시 콘텐츠 풀
- 몹 종류: **80종**
- 직업 (닉네임 기반 동적 외에 정해진 트리): **15종**
- 스킬: **120개** (직업×레벨 매트릭스)
- 장비: **300개**
- NPC: **40명** (이름·역할·기본 대사 사전 작성)
- 메인 스토리 챕터: **3개** (각 1~2시간 분량)
- POI: **출시 도시당 200곳** (서울 1차)

### 9.3 시즌 시스템
- 3개월 = 1 시즌
- 시즌마다 새 몹 20종, 신규 보스 1체, 한정 외형, 시즌 패스 출시
- 시즌 종료 시 캐릭터는 유지, 시즌 한정 보상은 종결

## 마치며

이 게임의 진짜 디자인 결정은 **"스태미나는 시간을 돈으로 환전하는 기계"**라는 것입니다. 시간이 없는 유저는 결제로 사고, 시간이 있는 유저는 광고로 사고, 둘 다 없는 유저는 내일을 기다리며 게임에 묶여 있습니다. LLM 비용이라는 부담을 게임의 가장 강력한 리텐션 메커니즘으로 변환하는 것 — 이게 LLM 시대 모바일 게임의 정답에 가장 가깝다고 생각합니다.

작성을 마치며, 이 기획서가 종이로 끝나지 않고 90초 후 누군가의 화면에서 "잿빛고양이가 골목에서 너를 본다" 라는 문장이 타이핑되기를.
